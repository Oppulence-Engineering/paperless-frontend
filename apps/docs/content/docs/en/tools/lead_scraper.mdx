Lead Scraper lets your workflows create scraping jobs, query leads, manage lead lists, and run saved searches. The service is multi-tenant:

- Organization: maps to your workspace ID
- Tenant: account created in Lead Scraper for the current user

You will need an API key to authenticate requests.
{/* MANUAL-CONTENT-END */}


## Usage Instructions

Provide `apiKey` for every operation. `organizationId` should map to the workspace ID, and `tenantId` is the Lead Scraper tenant/account ID created during provisioning (when omitted, the tool will fall back to workflow context or workspace environment variables when available). Use `baseUrl` and `apiPrefix` to target a self-hosted Lead Scraper gateway (for example `http://localhost:8081` and `/lead-scraper-microservice/api/v1`). Inputs named `request`, `update`, or `query` accept either a JSON object or a JSON string.

## Common Parameters

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | Lead Scraper API key |
| `organizationId` | string | No | Lead Scraper organization/workspace ID (defaults to workspace context) |
| `tenantId` | string | No | Lead Scraper tenant or account ID (defaults to workspace/user context) |
| `baseUrl` | string | No | Override Lead Scraper base URL |
| `apiPrefix` | string | No | Override Lead Scraper API prefix |

## Common Output

All Lead Scraper tools return the same envelope:

| Field | Type | Description |
| ---- | ---- | ----------- |
| `data` | json | Lead Scraper API response payload |
| `status` | number | HTTP status code |
| `headers` | json | Response headers |
| `cost` | json | Billing metadata (optional, when configured) |
| `quota` | json | Quota snapshot (optional, when available) |
